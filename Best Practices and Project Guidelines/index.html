HTML

Use proper Doctype

Doctype is a must for every webpage. And it should be first thing in the page source above <html> and NOTHING should be above it, be any HTML markup, CSS rules, JS code or any comments. Otherwise, the browser will render the page in quirks mode and may cause some unexpected issues.

For HTML5, which is the current standard for web pages, use <!DOCTYPE html>

Additionally, every page should have <html>, <head>, <body> & <title> tags.

Put CSS at the end of the <head> tag

Putting CSS styles and external stylesheets in <head> makes the page load faster.

Put JS at the end of <body> tag

Putting the external scripts in the end ensures that page rendering is not blocked due to loading of a script.

CSS

Keep your code clean & clear

Strictly maintain proper indentation and spacing for the selectors and their rules.

One indentation = 1 tab/4 spaces, no more, no less.

Remove unwanted whitespaces/new lines.

eg.

Avoid putting everything in a single line

Writing each selectors and each declarations in separate lines promotes readability and makes it easier to debug.

Make efficient use of selectors

Keep specificity to a minimum

#id: 100, .class: 10 tag: 1

Avoid using #id

Id has a specificity of 100, so if one wants to override this selector, he/she is forced to use id again increasing the complexity and decreasing the performance.

Avoid using !important

Using !important is always a bad idea. Try increasing specificity to achieve the same result by adding classes or tags. Id should be introduced only if absolutely necessary.

For already developed project, !important can only be accepted if the property to be overridden has already !important in place. Although, one must always try to remove the !important from existing code as well if it will not cause any regression.

Use shorthand properties whenever possible

Use vendor prefix for newer CSS3 properties

Adding vendor prefixes will support for a wide range of browsers and devices.

Reuse styles from other rules

Make use of certain global classes

Take advantage of frameworks such as Bootstrap

Decrease redundancy by merging selectors which have similar rules

Reduce the number of image requests to the server

Use sprites - add smaller images/icons in a sprite

Make sure an individual image request is sent only once per page.

B. Bad Practices and Better Design

Avoid using global variables

Variables declared outside of a function are attached to the global scope. Additionally, variables used without decalring it first, i.e. without using var keyword, whether it is inside or outside any function makes it a global variable.

Global variables are extremely risky as they can used or modified by any part of the code or any third party's code, either intentionally or un-intentionally.

Always declare the variables locally by using var keyword. Also, namespacing, closures, self-executing immediate functions are helpful in resolving clashing.

Proper use of var

Decalre all of the variables at the starting of the functions and preferably assign some initial value. Hoisting?

Avoid redeclartions of the variables

Never declare variables inside a block

Single var vs multiple var statements

Always use === instead of ==

Type checking

string - boolen - attention on number and stirng - null undefined

eval() === evil()

Never use eval() as it execute the string raising some security issues.

Never pass string in setTimeout or setInterval

Passing string in setTimeout or setInterval works in the sense as eval(). Always pass function as the parameter.

Bettern assignment

Use shorthand variable decalration(term??) wherever possible. Avoid using the new operator for primitive types, viz. String, Boolean, Object, etc.

Use ternary operator for assignment.

C. Performance

Major pain areas of performance impact

  • Page Rendering
  • Looping
  • DOM manipulation
  • Event binding/firing
  • HTTP requests

Looping

Avoid looping where it is not necessary

Avoid doing repetitive/heavy tasks that can be done outside the loop.

DOM Caching

DOM manipulation is a heavy task and has a lot of performance impact. While selecting some DOM node which can be used later, make sure to Cache it. This will save time to re-select the DOM.Also, cache wisely.

Optimize the use of selectors. Start with id attribute if possible and then move your way down to class, tag and pseudo selectors.

Multiple id selector in a single query is useless. Also, query the id selector alone and chain other selectors .

Event Binding

jQuery stacks event binding. Event will fire for each of the stacked event handler.

Event binding should not be done in a loop, function or callbacks. As each time the function will be called event binding will be stacked.

HTTP requests

HTTP requests should be kept to a minimum as they have overhead of moving back and forth to server. This includes number of images, sttylesheets, js files and ajax calls.

Compressing and Concating css and js files, use of sprites should be done.

Browsers also helps by caching these files.

Reduce Redundant code

Modularize your code. Try to break a large function into multiple smaller functions.

Try to bundle similar set of instruction into a function which can be used on later wherever needed.

Take out similar code out of if-else block.
